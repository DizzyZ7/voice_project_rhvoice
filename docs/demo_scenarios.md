# Демонстрационные сценарии

## Сценарий 1. Локальный монолит

1. Запустить `python voice_command_service.py --once`.
2. Произнести команду `включи свет`.
3. Ожидаемый результат:
   - STT распознаёт фразу;
   - логика маршрутизации выбирает `turn_on_light`;
   - RHVoice озвучивает `Готово`;
   - в `logs/voice_service.log` появляется запись о выполнении команды.

## Сценарий 2. GUI-демо

1. Запустить `python voice_command_gui.py`.
2. Нажать кнопку `Старт`.
3. Произнести `какая температура`.
4. Ожидаемый результат:
   - статус GUI меняется на `Распознано: какая температура`;
   - воспроизводится ответ `Сейчас температура 23.5 градусов`;
   - в журнале интерфейса фиксируется сценарий.

## Сценарий 3. Микросервисный контур

1. Запустить `docker compose up --build`.
2. Отправить WAV в orchestrator:
   `curl -X POST http://localhost:8002/process -H "Authorization: Bearer <VOICE_API_TOKEN>" -F "file=@sample.wav"`
3. Ожидаемый результат:
   - orchestrator вызывает STT и TTS;
   - при успешном распознавании публикуется MQTT-команда `factory/light/on`, `factory/light/off` или `factory/temperature/request`;
   - Prometheus получает метрики на `9101`, `9102`, `9103`;
   - Grafana отображает p95 latency и счётчики команд.

## Сценарий 4. Серия измерений

1. Запустить `python -m app.cli.benchmark`.
2. При наличии RHVoice дополнительно формируются WAV-файлы в `reports/tts/`.
3. Ожидаемый результат:
   - создаётся `reports/stage2_measurements.json`;
   - в отчёте видны точность на контрольном наборе фраз, latency, CPU и peak memory;
   - результаты можно включить в презентацию и финальный отчёт.

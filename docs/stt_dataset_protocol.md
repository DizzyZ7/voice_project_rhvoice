# Протокол подготовки STT-датасета для WER

## 1. Цель

Собрать репрезентативные WAV-кейсы для измерения `WER` в условиях, близких к эксплуатации:

- длинные оповещения (10+ слов);
- реальные промышленные шумы;
- вариативность дикторов и микрофонов.

## 2. Формат аудио

- `mono`
- `16 kHz`
- `PCM 16-bit`
- длительность 2-12 секунд
- один файл = одна контрольная фраза

## 3. Шумовые условия

На каждую ключевую фразу запишите минимум 4 варианта:

- тихий фон (40-50 dB);
- вентиляция / ровный цеховой фон (60-70 dB);
- шум оборудования (75-85 dB);
- импульсные помехи (удары/скрежет).

## 4. Структура файла кейсов

Создайте `benchmarks/stt_cases.json` по шаблону `benchmarks/stt_cases_template.json`:

```json
[
  {
    "wav_path": "benchmarks/noise_cases/example.wav",
    "expected_text": "зафиксировано превышение давления в реакторе номер три"
  }
]
```

`expected_text` должен быть нормализован (строчные буквы, без лишних пробелов).

## 5. Запуск измерения

```bash
python -m app.cli.benchmark --stt-cases benchmarks/stt_cases.json --output reports/stt_measurements.json
python scripts/check_quality_gates.py --report reports/stt_measurements.json
```

## 6. Рекомендуемые пороги (стартовые)

- `WER <= 25%`

Для production обычно целятся в более жёсткие значения после стабилизации словаря и микрофонного тракта.
